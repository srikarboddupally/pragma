Reciprocal Rank Fusion score(d) = Σ 1/(k + rank), k = 60

Scale-free: it uses only ranks, so incomparable dense-cosine and FTS-rank scores fuse without normalization. A chunk found by both retrievers accumulates two terms and rises.
  • #1Bgithub #82 — found by both (sparse#1 + dense#2)1/61 + 1/62 = 0.0325
  • #2Anotion policy — found by both (dense#1 + sparse#3)1/61 + 1/63 = 0.0323
  • #3Djira — sparse only1/62 = 0.0161
  • #4Cslack — dense only1/63 = 0.0159
↳ B and A — the two chunks both retrievers agreed on — top the fused list. Agreement is itself evidence of relevance.

Rerank window rerank_n = 20 · cross-encoder

A cross-encoder re-scores the fused top-N against the query directly (query+chunk together, not two independent vectors). Here it lifts B (the actual bug: “500 on rate-limit backoff”) above A. Fusion’s job was recall; the reranker’s job is precision.
  • #1Bgithub #82 — directly answers the failurererank 0.94
  • #2Anotion — explains expected behaviorrerank 0.88
  • #3Cslack — paraphrase of Arerank 0.86
  • #4Djirarerank 0.52

MMR — diversify argmax λ·rel(d) − (1−λ)·max_sim(d, selected) · λ = 0.7

C (slack) is a near-paraphrase of A (notion) — cosine ≈ 0.9. At λ=0.7 a duplicate must be (1−λ)/λ ≈ 0.43 more relevant to survive; C isn’t, so MMR drops it and keeps the diverse set. The top pick is never displaced (penalty is 0 with nothing yet selected).

Guardrail + action
A proposed action runs the five checks in order; the outcome is written to the append-only audit log.
action ▾
the audit row that gets written (append-only, on every outcome) 

      

      
Every action tool goes through guardrails.pipeline.evaluate() — never called directly — and every call requires a skill_id (the agent must fetch the skill before acting). The five checks are strictly ordered; an early failure short-circuits. skill match → condition eval → permission → risk tier → idempotency
